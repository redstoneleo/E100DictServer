# E600 客户端使用说明

本文档面向设备端/客户端开发，说明如何与服务端进行鉴权、使用 ChatBot（WebSocket 实时对话）和 VoiceToImage（语音转图片）功能。

"device\_secret": "381570a6a34e98ea6183687f0932ed938436323be216bab360f9ac2376c2edbe"==

==服务器网址：==[==https://e600.feing.com.cn==](https://e600.feing.com.cn/)

## 1. 术语与前置条件

*   `device_id`：设备唯一标识（服务端已注册）
*   `device_secret`：设备密钥，仅在注册时返回一次，需安全存储在设备端，不可明文上报
*   `nonce`：服务端下发的随机挑战串，缓存 60s，仅能使用一次
*   `signature/sign`：`HMAC_SHA256(device_secret, nonce)` 的十六进制小写字符串
*   `access token`：JWT，包含 `device_id` 等 claim，用于 API 鉴权

部署建议：

*   生产环境必须使用 `HTTPS/WSS`
*   服务端依赖缓存（通常是 Redis）存储 nonce 与用量统计

## 2. 统一鉴权流程（必须）

ChatBot 和 VoiceToImage 使用相同的 JWT Token + HMAC 签名认证机制，采用两步 challenge-response 方式，避免明文传输 `device_secret`。

### 2.1 获取 access/refresh token（Challenge-Response）

两步 challenge-response：

1.  请求 nonce（challenge）

`POST /device/challenge/`

请求：

```json
{ "device_id": "LYY-202603-000123" }
```

响应：

```json
{ "nonce": "..." }
```

1.  计算签名并换取 token

在设备端计算：

`signature = hex(HMAC_SHA256(key=device_secret, msg=nonce))`

`POST /device/token/`

请求：

```json
{ "device_id": "LYY-202603-000123", "signature": "<hex-hmac-sha256>" }
```

响应：

```json
{ "access": "<access_jwt>", "refresh": "<refresh_jwt>" }
```

补充：

*   nonce 在服务端缓存时间很短（约 60s），且成功使用后会被删除；过期或已使用需要重新获取 nonce
*   ChatBot WebSocket 连接需要额外的 nonce/sign（见 3.3）

### 2.2 刷新 access token（SimpleJWT 标准接口）

access token 过期后，使用 refresh token 换取新的 access：

`POST /api/token/refresh/`

请求：

```json
{ "refresh": "<refresh_token>" }
```

响应：

```json
{ "access": "<new_access_token>" }
```

## 3. ChatBot WebSocket 实时对话

### 3.1 连接地址

路径（服务端路由固定）：

*   `GET /ws/chatbot/`

连接 URL（示例）：

`wss://<host>/ws/chatbot/?token=<access>&nonce=<nonce_ws>&sign=<sign_ws>`

### 3.2 获取 WS 连接用的 nonce/sign（每次连接前都做一次）

在每次发起 WebSocket 连接之前，再调用一次：

`POST /device/challenge/` -> 得到 `nonce_ws`

在设备端计算：

`sign_ws = hex(HMAC_SHA256(key=device_secret, msg=nonce_ws))`

随后用 `access token + nonce_ws + sign_ws` 建立 WS 连接。

注意：`/device/token/` 在签发 token 成功后会删除该次 challenge 的 nonce（防重放）。因此建议按如下顺序做两次 challenge：

1.  获取 `token` 用的 challenge
2.  获取 WS 连接用的 challenge

### 3.3 关闭码（Close Code）

服务端会在握手后主动关闭（客户端应根据 code 做重试或提示）：

*   `4001`：未授权（token 无效 / nonce 不存在或过期 / sign 校验失败 / 设备不存在或被禁用）
*   `4002`：当日使用时长超限（见 3.7）

### 3.4 消息协议（客户端与服务端之间）

该 WS 同时使用两种 frame：

*   Binary frame：原始音频字节流
*   Text frame：JSON 控制/文本消息

#### 3.4.1 上行（客户端 -> 服务端）

1.  语音输入（binary）

*   客户端在"按住说话"期间，持续发送 binary frame
*   binary 内容为原始 PCM 音频字节（服务端声明为 `pcm16` 输入）
*   建议：小块发送（例如 20ms 到 100ms 一帧），避免单帧过大造成延迟抖动

1.  松开按键/停止说话（text）

当用户结束一轮说话（例如松开按钮）后，发送：

```json
{"action":"stop_speaking"}
```

服务端会提交输入缓冲并触发一次回答生成。

1.  打断当前回答（text）

当用户需要打断正在播报的回答并立即开始新一轮说话时，发送：

```json
{"action":"interrupt"}
```

服务端会取消当前回答并清空输入缓冲，随后你可以继续发送新的音频 binary frame。

#### 3.4.2 下行（服务端 -> 客户端）

1.  音频配置指令（text）

建立连接后，服务端会在首条控制消息中下发当前的下行音频采样率配置（因服务端可能切换 Qwen/BRTC 等不同引擎，输出采样率可能为 `24000` 或 `16000`）：

```json
{"action":"audio_config", "sample_rate": 24000}
```

客户端**必须**解析此消息，并在播放后面的 binary 音频时将底层的播放采样率调整为此值。

2.  TTS 音频输出（binary）

*   服务端会以 binary frame 方式持续下发回答音频 chunk，客户端应边收边播放。
*   音频帧的数据格式固定为 **16-bit 单声道小端 PCM** 原始字节流，**采样率**则由刚才的 `audio_config` 消息动态指定。

3.  文本转写增量（text）

服务端会下发 text frame（JSON）表示增量转写：

```json
{"action":"transcript","text":"..."}
```

说明：

*   `text` 为增量片段（delta），客户端可将其追加到当前显示文本上

### 3.5 推荐的客户端状态机

1.  连接前：确保已有 `access token`，并获取 `nonce_ws/sign_ws`
2.  建立 WS：连接成功后接收 `{"action":"audio_config", "sample_rate": ...}` 并配置本地播放流，进入 `IDLE`
3.  `IDLE -> RECORDING`：用户按住说话，开始周期性发送 PCM16 binary 帧（请注意：只有在完整传输所有录音 byte 后才能进入下一步，防止时序错位重启识别会话）
4.  `RECORDING -> WAITING`：用户松开按钮，发送 `{"action":"stop_speaking"}`，停止上行音频
5.  `WAITING -> PLAYING`：开始接收回答音频 binary 并播放，同时接收转写 text 并显示
6.  `PLAYING -> RECORDING`：若用户打断，先发送 `{"action":"interrupt"}`，停止播放，立刻进入录音并发送新音频
7.  任意状态断线：若 close code 为 `4001/4002` 按原因处理；否则重新获取 `nonce_ws/sign_ws` 后重连

### 3.6 用量限制与连接管理

服务端会按"连接持续时间"累计当日使用时长（不是纯说话时长）：

*   当日上限：`5 小时`（`5 * 60 * 60` 秒）
*   超限后将拒绝新连接（close code `4002`）

建议：

*   不用时及时断开 WS，避免空闲连接持续计费
*   断线重连前重新获取 `nonce_ws/sign_ws`，不要复用旧 nonce

### 3.7 示例代码（Python，演示握手与 WS 交互骨架）

下面示例只演示流程与消息格式；音频采集与播放请按你的平台实现。

```python
import requests, hmac, hashlib, json, asyncio, websockets

HOST = ""
BASE_HTTP = f"https://{HOST}"
BASE_WS = f"wss://{HOST}"

DEVICE_ID = "LYY-202603-000123"
DEVICE_SECRET = "<device_secret>"

def hmac_hex(secret: str, nonce: str) -> str:
    return hmac.new(secret.encode("utf-8"), nonce.encode("utf-8"), hashlib.sha256).hexdigest()

def get_nonce() -> str:
    r = requests.post(f"{BASE_HTTP}/device/challenge/", json={"device_id": DEVICE_ID}, timeout=10)
    r.raise_for_status()
    return r.json()["nonce"]

def get_tokens():
    nonce = get_nonce()
    sig = hmac_hex(DEVICE_SECRET, nonce)
    r = requests.post(f"{BASE_HTTP}/device/token/", json={"device_id": DEVICE_ID, "signature": sig}, timeout=10)
    r.raise_for_status()
    return r.json()["access"], r.json()["refresh"]

async def main():
    access, _refresh = get_tokens()

    # 每次 WS 连接前单独取一次 nonce/sign
    nonce_ws = get_nonce()
    sign_ws = hmac_hex(DEVICE_SECRET, nonce_ws)

    ws_url = f"{BASE_WS}/ws/chatbot/?token={access}&nonce={nonce_ws}&sign={sign_ws}"
    async with websockets.connect(ws_url, ping_interval=None) as ws:
        # 1) 发送若干音频块（二进制）。此处用假数据占位。
        # pcm16_bytes = capture_pcm16_chunk()
        # await ws.send(pcm16_bytes)

        # 2) 停止说话，触发回答
        await ws.send(json.dumps({"action": "stop_speaking"}))

        # 3) 接收：binary 为音频，text 为 json 控制命令
        while True:
            msg = await ws.recv()
            if isinstance(msg, bytes):
                # 按当前配置的采样率播放 play_pcm(msg)
                pass
            else:
                data = json.loads(msg)
                if data.get("action") == "audio_config":
                    # 例如重设采样率: set_sample_rate(data.get("sample_rate"))
                    print(f"Set sample rate to {data.get('sample_rate')}")
                elif data.get("action") == "transcript":
                    print(data.get("text", ""), end="", flush=True)

if __name__ == "__main__":
    asyncio.run(main())
```

## 4. VoiceToImage 语音转图片（HTTP API）

### 4.1 上传语音文件

```http
POST /voicetoimage/upload/
Content-Type: multipart/form-data
Authorization: Bearer {access_token}

参数：
- audio: 音频文件（必需）
- style: 图片风格（可选）：简笔画、线稿、素描、动漫、卡通、水墨
```

或者将 token 放在表单参数中：

```http
POST /voicetoimage/upload/
Content-Type: multipart/form-data

参数：
- audio: 音频文件（必需）
- style: 图片风格（可选）
- token: JWT access token（必需）
```

响应：

```json
{
    "status": "success",
    "task_id": 123,
    "message": "Voice recording uploaded and processing started."
}
```

### 4.2 查询任务状态

```http
GET /voicetoimage/status/{task_id}/
Authorization: Bearer {access_token}
```

或者使用 URL 参数：

```http
GET /voicetoimage/status/{task_id}/?token={access_token}
```

响应：

```json
{
    "id": 123,
    "status": "COMPLETED",
    "style": "卡通",
    "recognized_text": "识别的文字内容",
    "generated_image_url": "https://...",
    "error_message": null,
    "created_at": "2026-03-13T10:00:00Z"
}
```

任务状态说明：

*   PENDING: 等待处理
*   PROCESSING\_STT: 正在进行语音识别
*   PROCESSING\_IMAGE: 正在生成图片
*   COMPLETED: 已完成
*   FAILED: 失败

### 4.3 Python 客户端示例

```python
import requests
import hmac
import hashlib
import time

class VoiceToImageClient:
    def __init__(self, base_url, device_id, device_secret):
        self.base_url = base_url
        self.device_id = device_id
        self.device_secret = device_secret
        self.access_token = None
        self.refresh_token = None
    
    def authenticate(self):
        """两步 challenge-response 获取 token"""
        # 1. 请求 nonce
        response = requests.post(
            f"{self.base_url}/device/challenge/",
            json={"device_id": self.device_id}
        )
        response.raise_for_status()
        nonce = response.json()['nonce']
        
        # 2. 计算签名
        signature = hmac.new(
            self.device_secret.encode('utf-8'),
            nonce.encode('utf-8'),
            hashlib.sha256
        ).hexdigest()
        
        # 3. 换取 token
        response = requests.post(
            f"{self.base_url}/device/token/",
            json={
                "device_id": self.device_id,
                "signature": signature
            }
        )
        response.raise_for_status()
        data = response.json()
        self.access_token = data['access']
        self.refresh_token = data['refresh']
        return data
    
    def refresh_access_token(self):
        """刷新 access token"""
        if not self.refresh_token:
            raise Exception("No refresh token available")
        
        response = requests.post(
            f"{self.base_url}/api/token/refresh/",
            json={"refresh": self.refresh_token}
        )
        response.raise_for_status()
        self.access_token = response.json()['access']
        return self.access_token
    
    def upload_voice(self, audio_file_path, style='卡通'):
        """上传语音文件"""
        if not self.access_token:
            self.authenticate()
        
        with open(audio_file_path, 'rb') as f:
            files = {'audio': f}
            data = {'style': style}
            headers = {'Authorization': f'Bearer {self.access_token}'}
            
            response = requests.post(
                f"{self.base_url}/VoiceToImage/upload/",
                files=files,
                data=data,
                headers=headers
            )
            
            # 如果 token 过期，尝试刷新后重试
            if response.status_code == 401:
                self.refresh_access_token()
                headers = {'Authorization': f'Bearer {self.access_token}'}
                f.seek(0)  # 重置文件指针
                response = requests.post(
                    f"{self.base_url}/VoiceToImage/upload/",
                    files={'audio': f},
                    data=data,
                    headers=headers
                )
        
        response.raise_for_status()
        return response.json()
    
    def get_task_status(self, task_id):
        """查询任务状态"""
        if not self.access_token:
            self.authenticate()
        
        headers = {'Authorization': f'Bearer {self.access_token}'}
        response = requests.get(
            f"{self.base_url}/VoiceToImage/status/{task_id}/",
            headers=headers
        )
        
        # 如果 token 过期，尝试刷新后重试
        if response.status_code == 401:
            self.refresh_access_token()
            headers = {'Authorization': f'Bearer {self.access_token}'}
            response = requests.get(
                f"{self.base_url}/VoiceToImage/status/{task_id}/",
                headers=headers
            )
        
        response.raise_for_status()
        return response.json()

# 使用示例
if __name__ == '__main__':
    client = VoiceToImageClient(
        base_url='http://127.0.0.1:8000',
        device_id='FA1-202603-000123',
        device_secret='381570a6a34e98ea6183687f0932ed938436323be216bab360f9ac2376c2edbe'
    )
    
    # 上传语音
    result = client.upload_voice(r'F:\BaiduNetdiskDownload\ConsumerSoftwareProject\E100DictServer\media\voice_recordings\书包.pcm', style='卡通')
    print(f"Upload result: {result}")
    
    if result['status'] == 'success':
        task_id = result['task_id']
        
        # 轮询查询状态
        while True:
            status = client.get_task_status(task_id)
            print(f"Task status: {status['status']}")
            
            if status['status'] in ['COMPLETED', 'FAILED']:
                print(f"Final result: {status}")
                if status['status'] == 'COMPLETED':
                    print(f"Recognized text: {status['recognized_text']}")
                    print(f"Image URL: {status['generated_image_url']}")
                else:
                    print(f"Error: {status['error_message']}")
                break
            
            time.sleep(2)
```

## 5. 错误码说明

*   400: 请求参数错误（缺少音频文件等）
*   401: 认证失败（token 无效或过期）
*   403: 权限不足（设备未激活或访问其他设备的任务）
*   404: 资源不存在（设备或任务不存在）
*   405: 请求方法错误

## 6. 常见问题与排查

### ChatBot WebSocket

*   连接后立即被断开且 code=4001：
    *   检查 `token/nonce/sign` 三者是否都带上了
    *   确认 `nonce_ws` 是刚从 `/device/challenge/` 获取且未过期（60s）且未被复用
    *   确认 `sign_ws` 算法是 `HMAC_SHA256(device_secret, nonce_ws)`，输出 hex
*   返回 `challenge expired or not found`：
    *   nonce 过期或已被消费，重新调用 `/device/challenge/`
*   无法正常播放音频（或声音发扁/变调/滋啦滋啦有杂音）：
    *   服务端下发的音频为 **16-bit 单声道小端 PCM** 原始流，且**采样率不再固定**。建立连接时服务端会通过 `{"action": "audio_config", "sample_rate": ...}` 动态下发（Qwen 通常为 24000，BRTC 通常为 16000）。
    *   客户端的底层硬件/框架解码播放时，**必须**挂载对应下发的采样率，否则会出现倍速、慢放以及“滋啦滋啦”的严重断顿与爆音。
    *   上行（客户端发往服务端）的语音格式要求保持不变：16000 Hz、16-bit、单声道、小端。

### VoiceToImage HTTP API

*   只能查询自己设备创建的任务，无法访问其他设备的任务
*   音频文件格式建议使用 WAV、MP3 等常见格式

## 7. 安全建议

1.  妥善保管 `device_secret`，不要硬编码在客户端代码中
2.  使用 HTTPS/WSS 加密传输
3.  定期轮换 refresh token
4.  监控异常的 API 调用频率
5.  使用两步 challenge-response 方式获取 token，`device_secret` 永远不会在网络上传输
6.  Nonce 在服务端缓存时间很短（约 60s），且成功使用后会被删除
