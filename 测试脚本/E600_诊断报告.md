# E600 服务器 WebSocket 诊断报告

## 测试结果总结

### ✅ 成功的部分
1. **HTTP 认证端点工作正常**
   - `/device/challenge/` - 200 OK
   - `/device/token/` - 200 OK
   - 成功获取 access token 和 refresh token

2. **本地代码配置正确**
   - WebSocket 路由: `/ws/chatbot/`
   - ASGI 配置正确
   - Channels 配置正确

### ❌ 失败的部分
1. **WebSocket 连接返回 404**
   - 路径: `wss://e600.feing.com.cn/ws/chatbot/`
   - 错误: HTTP 404 Not Found

## 问题分析

WebSocket 返回 404 说明服务器上的 WebSocket 路由没有正确配置。可能的原因：

### 1. ASGI 服务器未运行
服务器可能只运行了 WSGI（如 Gunicorn），而没有运行 ASGI 服务器（如 Daphne 或 Uvicorn）。

**检查方法：**
```bash
# 在服务器上检查进程
ps aux | grep -E "daphne|uvicorn"
```

### 2. Nginx/Apache 配置问题
Web 服务器可能没有正确配置 WebSocket 代理。

**Nginx 正确配置示例：**
```nginx
location /ws/ {
    proxy_pass http://127.0.0.1:8001;  # ASGI 服务器地址
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 86400;
}
```

### 3. 防火墙或安全组规则
WebSocket 端口可能被防火墙阻止。

### 4. Django Channels 未安装或配置
服务器上可能缺少必要的依赖。

## 解决方案

### 方案 1: 部署 ASGI 服务器

#### 使用 Daphne（推荐用于 Channels）

1. **安装依赖**
```bash
pip install daphne channels channels-redis
```

2. **启动 Daphne**
```bash
daphne -b 0.0.0.0 -p 8001 E100DictServer.asgi:application
```

3. **使用 Systemd 管理（生产环境）**

创建 `/etc/systemd/system/daphne.service`:
```ini
[Unit]
Description=Daphne ASGI Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/path/to/E100DictServer
Environment="DJANGO_SETTINGS_MODULE=E100DictServer.settings"
ExecStart=/path/to/venv/bin/daphne -b 127.0.0.1 -p 8001 E100DictServer.asgi:application
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl start daphne
sudo systemctl enable daphne
```

#### 使用 Uvicorn（替代方案）

```bash
pip install uvicorn[standard]
uvicorn E100DictServer.asgi:application --host 0.0.0.0 --port 8001
```

### 方案 2: 配置 Nginx

在 Nginx 配置文件中添加 WebSocket 代理：

```nginx
server {
    listen 443 ssl;
    server_name e600.feing.com.cn;

    # SSL 配置
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # HTTP 请求（Django WSGI）
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket 请求（Django ASGI）
    location /ws/ {
        proxy_pass http://127.0.0.1:8001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }
}
```

重新加载 Nginx：
```bash
sudo nginx -t
sudo systemctl reload nginx
```

### 方案 3: 检查依赖

确保服务器上安装了所有必要的包：

```bash
pip install -r requirements.txt
```

**requirements.txt 应包含：**
```
Django>=4.2
channels>=4.0
channels-redis>=4.0
daphne>=4.0
djangorestframework>=3.14
djangorestframework-simplejwt>=5.3
redis>=5.0
websockets>=12.0
```
